# Homework-8-Deployment-ML

## Описание проекта

В рамках работы была собрана система мониторинга ML-сервиса с использованием Prometheus и Grafana.  
Также были рассмотрены задачи обнаружения деградации модели, дрифта данных, а также контроля качества данных.  
Дополнительно была спроектирована архитектура системы виртуального встраивания рекламы в видеопоток.

Проект выполнялся в Google Colab без использования Docker.

---

## Что реализовано

### 1. Мониторинг ML-сервиса

Был развёрнут ML-сервис с метриками `/metrics`.  
С помощью Prometheus собирались следующие метрики:
- задержка обработки запросов (request_latency_seconds)
- количество запросов

В Grafana был построен дашборд с отображением p95 latency.

---

### 2. SLO

Для системы были определены следующие SLO:
- p95 latency меньше 1 секунды
- уровень ошибок меньше 1%
- доступность сервиса выше 99%

---

### 3. Алертинг

В Grafana настроено правило алерта:
- если p95 latency превышает 1 секунду в течение 2 минут, срабатывает предупреждение

---

### 4. Дрифт данных

Было выполнено сравнение:
- reference_data (эталонный набор)
- current_data (изменённый набор)

Для определения дрифта использовался статистический тест KS-test.  
Был искусственно изменён входной набор данных для демонстрации смещения распределений.

---

### 5. Контроль качества данных

В PostgreSQL была создана таблица users.  
Далее была изменена её структура, что привело к нарушению согласованности данных.

Это позволило зафиксировать инцидент качества данных.

---

### 6. Архитектура ML-системы

Была спроектирована архитектура системы виртуального встраивания рекламы в видеопоток.

Выбрана Kappa-архитектура, так как система работает в режиме потоковой обработки.

Схема обработки:
видеопоток → брокер сообщений → модель детекции объектов → генеративная модель → хранилище → API
